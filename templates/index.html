Cloudinary Configuration

Video tải qua API /api/download sẽ được upload lên Cloudinary rồi xoá khỏi server ngay — cần nhập thông tin tài khoản Cloudinary bên dưới trước khi dùng endpoint đó.

Đang dùng biến môi trường (CLOUDINARY_CLOUD_NAME/API_KEY/API_SECRET) đặt sẵn trên server — không thể sửa qua giao diện.

Nếu không đặt biến môi trường, giá trị nhập ở đây được lưu vào file trên server và sẽ mất khi Render redeploy — cần nhập lại sau mỗi lần deploy code mới.

API cho website / server khác

Web hoặc server khác có thể gọi thẳng API này để tải video — không cần dùng giao diện. Dùng API key ở góc trên bên phải trang này (bấm biểu tượng mắt để xem, biểu tượng copy để sao chép).

Kèm sẵn API key thật + luồng đăng ngay/lên lịch — gửi thẳng cho dev bên kia.

Nên gọi từ backend/server của bạn, không gọi trực tiếp từ JavaScript chạy trên trình duyệt người dùng cuối — key sẽ bị lộ ra source code phía client.

Base URL
https://autodown.vibevic.com
Header xác thực (bắt buộc cho mọi request /api/*)
X-API-Key: <API_KEY_CUA_BAN>

Extract metadata — POST /api/extract

Không tải file, không upload Cloudinary — chỉ lấy caption/thumbnail để hiển thị trước.

curl -X POST https://autodown.vibevic.com/api/extract \
  -H "X-API-Key: <API_KEY_CUA_BAN>" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.facebook.com/reel/XXXXXXXXXX"}'

# → {"success": true, "platform": "facebook", "type": "video", "caption": "...", "thumbnail": "...", "mediaCount": 1}

Tải + upload Cloudinary (1 lần gọi) — POST /api/download

Tải video, upload lên Cloudinary, xoá file tạm trên server ngay sau khi upload xong, trả về link Cloudinary. Cần cấu hình Cloudinary ở mục phía trên trước. Request có thể mất vài giây tới vài chục giây tuỳ độ dài video. 

curl -X POST https://autodown.vibevic.com/api/download \
  -H "X-API-Key: <API_KEY_CUA_BAN>" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.facebook.com/reel/XXXXXXXXXX"}'

# → {"success": true, "caption": "...", "type": "video",
#     "media": [{"type": "video", "url": "https://res.cloudinary.com/.../temp/abc123/xxxx.mp4", "public_id": "temp/abc123/xxxx"}]}

Xoá asset Cloudinary — POST /api/cleanup

Gọi khi web app của bạn không cần asset đó nữa — dùng đúng public_id nhận được từ /api/download.

curl -X POST https://autodown.vibevic.com/api/cleanup \
  -H "X-API-Key: <API_KEY_CUA_BAN>" \
  -H "Content-Type: application/json" \
  -d '{"public_ids": ["temp/abc123/xxxx"]}'

# → {"success": true, "deleted": {"temp/abc123/xxxx": "deleted"}}

Service này không lưu job/state gì cả — mỗi request tự chứa toàn bộ thông tin cần thiết. Web app gọi API chịu trách nhiệm giữ lại public_id và tự gọi /api/cleanup khi xong việc.

1. Bắt đầu tải (luồng polling cho giao diện web) — POST /api/start_dl

Body JSON: url (bắt buộc), height (tuỳ chọn, null = chất lượng cao nhất).

curl -X POST https://autodown.vibevic.com/api/start_dl \
  -H "X-API-Key: <API_KEY_CUA_BAN>" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://www.facebook.com/reel/XXXXXXXXXX", "height": null}'

# → {"dl_id": "a1b2c3d4e5f6"}

2. Kiểm tra trạng thái — GET /api/dl_status/<dl_id>

Gọi lặp lại (poll) mỗi 1–2 giây cho tới khi status là "done" hoặc "error".

curl https://autodown.vibevic.com/api/dl_status/a1b2c3d4e5f6 \
  -H "X-API-Key: <API_KEY_CUA_BAN>"

# → {"status": "done", "percent": 100, "filename": "caption_video.mp4", "caption": "...", "error": null}

3. Tải file — GET /api/dl_file/<dl_id>

Chỉ gọi được sau khi status = "done". File chỉ tồn tại trên server trong ít phút rồi bị xoá — tải về ngay khi xong.

curl https://autodown.vibevic.com/api/dl_file/a1b2c3d4e5f6 \
  -H "X-API-Key: <API_KEY_CUA_BAN>" \
  -o video.mp4

Nền tảng hỗ trợ: Facebook, TikTok, YouTube (chỉ video/reel công khai). Không hỗ trợ ảnh/album. App không bao giờ dùng cookie đăng nhập cho bất kỳ nền tảng nào.

Chỉ dùng cho video công khai hoặc video của bạn.
